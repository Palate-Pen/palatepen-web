Prep v1 — Design Notes
New top-level chef tab — the missing kitchen-floor surface. Prep is what kitchen-floor chefs actually live in day-to-day: what's getting made, by whom, in what quantity, status now. Without this, every other surface's references to prep (waste analysis, margin attribution, delivery planning) are hollow.

Day-anchored, station-grouped. Today is the default view, with horizontal day-tabs to navigate forward/back. Each station gets its own header with assigned chef name and progress summary. Tom's Garde Manger, Maria's Grill, Sam's Pass, Pastry unassigned today.

Covers context strip sits above the stations. Tonight's 142 booked + 156-168 forecast, with the system's read: "tracking 18% above last Thursday, hummus and shawarma scaled accordingly." This is how covers data informs prep — not by auto-setting quantities, but by giving the chef context to set them well.

Each prep row: item · quantity · assigned · status · notes. The quantity column shows chef-set value with a muted "suggested" hint beneath — system suggests, chef decides. Suggested can flag attention (e.g. "cut back 20% — last 4 weeks binned £62") but never overrides. Status pills follow the design system pattern (Done healthy, In Progress gold, Not Started muted, Over-Prepped attention, Short urgent).

Recipe-linked items show a recipe pill. Click-through to the recipe to scale, adjust, view ingredients. One-off items (parsley garnish, mezze plating mise) are tagged as one-off.

Notes column captures the chef's running thoughts — "2% brine this time, not 3%" / "Used labneh to thin per Tuesday's voice memo". These are the prep-day equivalent of notebook entries — quick text, sometimes referencing other surfaces.

Looking Ahead does forward-prep intelligence. Friday's covers tracking high (scale shawarma + tahini, sea bream needs assignment) and weekend over-prep pattern (scale Saturday down 15%). Connects covers forecasting + waste patterns into actionable prep planning — exactly what prep needs to be informed by.

Critically: the chef sets the prep. System suggests based on covers × recipes × past patterns. System flags when something looks off vs history. System never auto-overrides. Prep is chef work, and the surface reflects that.
