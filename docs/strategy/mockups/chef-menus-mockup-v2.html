Menus v2 — Edit-Light Rationale
Why edit-light. The manager menu builder handles structural changes: section creation, layout templates, menu engineering overlays. The chef doesn't need that ceremony on a Tuesday afternoon. Edit-light covers the three operational moves a chef actually makes mid-service or on the day: change a sell price, 86 a dish for tonight, reorder within a section. That's it. Structural work stays one click away via "Open in Menu Builder →".

The mode toggle. View / Edit / Planning sits inline at the top. View is the existing read-only summary surface. Planning hosts the Kasavana & Smith matrix and forward menu work — chef-led, but separate context. Edit is this surface. The toggle is a query param (?mode=edit) not a separate route, so the data + URL share state.

Staging and saving. Edits stage immediately in local state — the row goes gold, GP recomputes live, the delta is shown ("+£2 → +1pt"). Nothing is written to Postgres until the chef hits Save changes on the sticky bar. Discard reverts the local diff. Save is per-batch — if a price moves twice, only the final value lands. The bar shows "3 unsaved changes" in the page subtitle and in the bar itself, so the chef can't accidentally leave staged work behind.

86 tonight. Distinct from "remove from menu". 86 is for today's service — the dish stays on the structural menu, stays priced, but won't print on tonight's customer copy and is flagged for the POS as unavailable. Resets at midnight unless the chef extends it. This is the move chefs care about: "we ran out of lamb shoulder, hide the shawarma until Thursday's delivery". The full "remove from menu" path is a manager builder action.

Add dish flow. Adding a dish to a section is "promote an existing recipe", not "create a new dish". The dialog filters Recipes by section, shows un-priced ones first, and surfaces "no sell price yet" recipes. Chef picks one, adds a price, save. New dish creation still lives at /recipes/new — different mental task.

Reorder. Drag handle (⋮⋮) at row right. Chef can reorder within a section. Cross-section moves go through Menu Builder.

GP recompute. Live, per-row, against the recipe's current cost (from The Bank). If the chef adjusts a price upward, GP rises and the delta chip shows the impact. If The Bank's cost moved overnight, the GP already reflects that — no manual refresh.
