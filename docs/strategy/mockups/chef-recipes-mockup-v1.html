Recipes v1 — Costing Inline
One tab, two functions: recipes + costing merged. Each dish lives as a tile. Header shows dish name + prep yield. Cost bar shows cost per cover (green healthy colour) + total dish cost. Ingredients list shows qty, source ingredient name from The Bank, and live cost pulled from current Bank price. If tahini is £12/L, the cost updates instantly across all recipes using tahini.

Three actions per tile: Edit · Scale · Print. Edit opens the recipe detail view (not shown). Scale lets the chef adjust cover count on the fly (*"if we're 180 covers tonight instead of 142, scale shawarma up to 2.2 portions"*). Print gives a kitchen print-out of the recipe with current costs embedded.

Cost is the intelligence layer. Chef sees instantly which dishes are expensive (beef short rib £6.84 per cover) vs efficient (hummus £1.24). Over time, patterns emerge — *"our margin on knafeh is tight at current honey prices, consider a lemon posset alternative when honey spikes"* — that surface in Looking Ahead on the manager dashboard.
