01 · File Types

What we accept, and what we don't.

Palatable supports the four formats designers actually work in: PDF (vector with editable text), AI (Adobe Illustrator), Figma (exported as .fig or shared via link), and PSD (Photoshop). Max file size is 20MB.

Accepted
PDF — vector preferred, with editable text layers (not flattened or outlined)
AI — Illustrator CC 2020+ format, with named layers
Figma — paste a share link with view access, or export as .fig
PSD — works but text-heavy menus do better in vector formats
Won't work
Flattened PDFs (text rasterised to image), JPG/PNG only (no text data), InDesign INDD files (use PDF export), Word/Pages documents (export to PDF first). If your file is older than 2020 or you're not sure if text is editable, open it in Acrobat or Preview and try to highlight a dish name — if you can't select text, we can't inject it.
02 · Page Sizes

Page sizes and safe zones.

We support standard menu sizes out of the box: A4 portrait (210 × 297mm), A5 portrait (148 × 210mm), US Letter (8.5 × 11in), tri-fold (varies), and custom sizes up to A2. Make sure your file uses one of these consistently.

Header
Dishes
Footer

Three zones we recognise

Header zone — your logo, menu title, "Dinner Menu" eyebrow, location. Stays static between updates.

Dishes zone — where Palatable injects content. Leave this empty in your template, or fill with placeholder text we'll replace.

Footer zone — allergen info, dietary notes, social handles, address. Stays static.

Use bleed and trim guides as you would for print. Keep critical content 5mm inside the trim line.

03 · Layer Naming

Name your layers semantically.

This is the most important step. We read your layer/frame names to know what goes where. Use these exact prefixes — case-insensitive — and Palatable will inject content into the right places automatically.

Layer / Frame Name
What Palatable Injects
menu-title
The menu name you set in Palatable ("Dinner Menu — May 2026")
menu-eyebrow
Optional eyebrow text above the title ("Berber & Q · Shoreditch")
menu-sub
Optional sub-headline beneath the title
section-header-N
N is the section index (1, 2, 3...). Maps to "Starters", "Mains", "Desserts" in order.
dish-row-N-M
N = section, M = dish. Container for one dish (name + description + price grouped).
dish-name
Dish name (must be inside a dish-row container)
dish-desc
Dish description (optional — toggle on/off in Palatable)
dish-price
Price — Palatable updates this when you change the price
footer-allergens
Allergen disclaimer text (we don't modify, just preserve)
Tip
If you have 15 dishes across 3 sections, you'll have section-header-1, section-header-2, section-header-3, and dish rows numbered dish-row-1-1 through dish-row-3-5. Make as many dish rows as your menu's longest section. Empty rows hide automatically.
04 · Editable Text Frames

Use real text frames, not paths.

Every text element we need to update must be a live text frame — not text converted to outlines or paths. If your designer converted text to paths for typographic effect, Palatable can't edit it. Keep at least the dish-name, dish-desc, dish-price, and section-header frames as live text.

Text frame setup
Single-line frames for dish-name and dish-price. Auto-resize disabled.
Auto-height multi-line frames for dish-desc. We expand as needed without breaking layout.
Paragraph styles applied — we read the style, not inline overrides
Right-aligned for dish-price so currency symbol and number stay visually anchored
05 · Fonts

Embed fonts or use web-safe alternatives.

Custom fonts must be embedded in PDFs (subset is fine) and linked correctly in AI/PSD. For web menu output, we need Google Fonts equivalents — or upload your font files as a separate ZIP if you have web-licensed versions. Brand fonts without web licenses will fall back to closest Google Font.

License check
Most premium fonts are licensed for print but not for web embedding. If your brand uses Helvetica Neue, GT America, or other commercial fonts, check with your designer whether you have web licenses. Without them, your web menu and QR code menu will use a Palatable-selected Google Font alternative. Your print PDF will use your original fonts.
06 · Colour Profiles

CMYK for print, sRGB for web.

Palatable auto-converts colour profiles depending on output, but for best print accuracy, supply your file in CMYK with FOGRA39 or US SWOP profile. We'll generate sRGB versions for the web menu and QR output automatically.

Spot colours (Pantone) are preserved in print output but converted to nearest CMYK for digital. If you've used a specific brand Pantone, supply the CMYK breakdown alongside the file.

07 · Images and Placeholders

If you use dish photography.

Most editorial menus don't use dish photos, but if yours does, place image frames using these naming conventions and we'll inject the correct dish photo from your Palatable library:

Frame Name
What Palatable Injects
dish-image-N-M
Image for the dish in section N, position M. Pulls from Recipes library photo.
logo
Your kitchen logo (uploaded once in Palatable Settings → Brand)
hero-image
Optional hero image at top of menu

Supply image frames sized to the final dimension you want — Palatable will fit dish photos to the frame using a smart-crop algorithm that prioritises plate-centring.

08 · What Palatable Injects vs Preserves

Your design stays yours.

We only inject content into the layers you've named. Everything else — your logo placement, decorative elements, illustrations, borders, hand-drawn flourishes, custom typography effects, paper texture, layout grid — is preserved exactly as you designed it.

What Palatable updates automatically
Dish names, descriptions, and prices from your Recipes library
Section headers in the order you've arranged them in Menu Builder
Dish images (if frames named correctly)
Removes empty dish rows automatically when sections are shorter
What Palatable never touches
Your logo, brand marks, decorative typography
Backgrounds, paper textures, illustrative elements
Footer copyright, allergens, social handles, address
Any layer you haven't named with our conventions
09 · Final Checklist

Before you upload.

Run through this list
File is PDF, AI, Figma, or PSD — under 20MB
Page size is A4 / A5 / US Letter / tri-fold / custom (max A2)
All layers named semantically (menu-title, section-header-N, dish-row-N-M, etc.)
Text frames are live (not converted to paths)
Fonts embedded or web-licensed alternatives noted
Colour profile is CMYK for print files
Image frames named correctly if using dish photography
Header and footer zones identified clearly
