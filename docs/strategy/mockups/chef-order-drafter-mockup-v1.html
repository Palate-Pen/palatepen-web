Order Drafter v1 — Design Notes
Modal/sheet surface, not a page. Launches over the current surface (Deliveries scheduled row clicked "Draft order", or Suppliers detail page "Place order"). Dark scrim, drafter card centred with shadow. Closes back to the surface that launched it.

Three sections, vertical flow: Order Items (the what), How to Send (the channel), Your Email (the composed message). Each section labelled in Cinzel with an italic meta line giving context — "Suggested from your Saturday pattern" / "Sarah prefers WhatsApp · email for record" / "Edit anything before sending".

Items table with editable quantities. Each row: item · qty input (+/− buttons) · unit · estimated value · remove. Estimated value pulled live from The Bank. Total at the bottom shows £162.90 with the system's "£175 ± 10% based on last 4 weeks" range as context — chef knows if their order is unusual.

Suggestion row inline. Gold-bg strip mid-table: "Friday's looking heavy — 168 covers booked. Last time your Saturday top-up included chicken wings for the weekend special. Worth adding?" + Add 4kg button. System suggesting based on Friday's covers + past Saturday patterns. One click adds the line.

Four-channel selector. Email selected (gold border, gold top bar), WhatsApp available, Portal disabled because Aubrey doesn't have one, Phone available (would just show the number with a "remember to log the order" reminder). Each card shows the channel-specific destination (email address, phone number, etc).

Composed message preview, fully editable. To/From/Subject in stacked fields with gold-bordered field labels. Body in Cormorant Garamond — reads like a chef writing to a supplier they know, not auto-generated. Greeting, the order as a clean bulleted list with item specs in bold, delivery slot reference, signature with phone number in muted text.

Footer commits the action. "Order will log as Expected delivery on Sat 17 May" tells the chef what happens next — the Deliveries row that previously said "not yet ordered" updates the moment they send. Primary action "Open in mail" launches the mailto: link with the composed message pre-filled. Save as draft for later.

The cross-surface loop: Deliveries shows scheduled order → click Draft order → drafter opens → chef edits + sends via Email → returns to Deliveries → row now reads "Expected · sent via email to Sarah" with a small email icon. Order logged, expected delivery tracked, paper trail intact.
